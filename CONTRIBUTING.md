## 🚀 Getting Started

To get started with contributing:

1. Fork the repository
2. Clone your fork
   git clone [https://github.com/Bavanetha27/Aira.git](https://github.com/Bavanetha27/Aira.git)
3. Navigate to the project directory
   cd Aira
4. Create a new branch
   git checkout -b feature/your-feature-name

## ⚙️ Development Setup

### Backend Setup

cd server
npm install
npm start

### Frontend Setup

cd client
npm install
npm run dev

## 🌿 Branching Strategy

Use clear and meaningful branch names:

* feature/feature-name → New features
* fix/bug-name → Bug fixes
* docs/update-docs → Documentation changes
* refactor/code-cleanup → Code improvements

---

## 📝 Commit Guidelines

Follow a consistent commit message format:

* feat: add health tracking feature
* fix: resolve login bug
* docs: update README
* refactor: improve API structure

Keep commits:

* Small
* Clear
* Meaningful

---

## 🔄 Pull Request Process

1. Ensure your branch is up to date
2. Push your changes
   git push origin your-branch-name
3. Open a Pull Request

### PR Description Should Include:

* What changes were made
* Why the changes were made
* Screenshots (if UI-related)

### PR Requirements:

* Code builds successfully
* No console errors
* Proper formatting followed
* Linked to an issue (if applicable)

---

## 🐛 Issue Guidelines

Before creating an issue:

* Check if the issue already exists
* Use a clear and descriptive title
* Provide detailed information
* Add screenshots if necessary

---

## 🎨 Code Style Guidelines

* Follow consistent formatting
* Use meaningful variable and function names
* Keep code modular and reusable
* Avoid unnecessary complexity
* Comment only when necessary
